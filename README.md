# CP1404 Assignment 1 - Books to Read

Assignment 1 for CP1404/CP5632, IT@JCU

By: Quoc Huynh

This repo contains starter files including a Python file and two CSV files.  
`books.csv` is a data file for you to use.  
`books_backup.csv` is the same data, but not to be used... it is here as a backup to copy the data from if you need it
again.

This program, Books to Read 1.0, is a Python application that allows users to keep track of books they have read and books 
they plan to read. The program loads book data from a CSV file, displays a neatly formatted list of books, and lets the user
add new books or mark existing ones as completed. It demonstrates the use of functions, loops, file I/O, and exception based
error handling. The project was developed closely following the CP1404 style guide for clean, modular, and readable code. 

1. How long did the entire project (assignment 1) take you?

It took me four days to complete the main functionality of the program. It took me another 4 days to fix up the minor
details and refactor all the functions for SRP so overall, assignment 1 took 8 days. 

2. What do you plan to do differently in your development process for assignment 2?

For assignment 2, I plan to make a clearer plan of what my program should look like before beginning to code it. This will
prevent me from needing to go back and fix bugs and restructure things later. I also plan to commit more frequently, and use
clearer commit messages to document each minor milestone.
